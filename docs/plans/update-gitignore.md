#### 1.2 Rationale for each section (reference only — do not include in file)

| `.npmrc` | Private AWS CodeArtifact registry auth token — must never be committed |

#### 1.3 Note on `*.local` and `settings.local.json`
The pattern `*.local` in git matches any file ending with `.local` anywhere in the repo tree. `.claude/settings.local.json` ends with `.json`, not `.local`, so `*.local` does NOT match it. No special negation rule is needed. Confirm this by noting: `settings.local.json` — the filename ends in `.json`. The `.local` pattern matches filenames whose full name ends in the literal string `.local` (e.g. `.env.local`, `vite.config.local`). This file is safe.

## 7. Risk and Rollback

### 7.1 Known Risks
- **Risk**: The `*.local` pattern matches any file ending in `.local`. If a developer creates a file like `config.local` in any subdirectory, it will be ignored automatically. This is intentional. **Mitigation**: Document this in team onboarding notes.
- **Risk**: `.claude/compacted/` ignore rule only ignores that one subdirectory. If Claude creates new machine-local cache subdirectories in the future, they must be added manually. **Mitigation**: Monitor `.claude/` for new untracked directories when upgrading Claude Code.
- **Risk**: `**/dist/` pattern is broad. If any future package needs to commit dist files (e.g., a vendored pre-built package), that package must add a local `.gitignore` negation or the root rule must be scoped. **Mitigation**: None needed currently — no `dist/` content is tracked anywhere in the repo.

### 7.2 Rollback Procedure
If the new `.gitignore` causes unexpected behavior:

1. Run: `git -C "D:/VNPT_ODA/poc-folder/ODA_COMPONENT/oda-components" checkout HEAD -- .gitignore`
2. Verify: `git -C "D:/VNPT_ODA/poc-folder/ODA_COMPONENT/oda-components" status` shows `.gitignore` is back to the last committed version.
