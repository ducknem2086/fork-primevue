## 1. Context Summary

### 1.1 Original Request
Build the entire `@oda-components/` component library project into a single, unified bundle file, avoiding fragmented output. The goal is to allow clean and centralized imports such as `import { OdaButton } from '@oda-components/components'`.

### 1.2 Why This Solution
Vite Library Mode is chosen because Vite 8 and `@vitejs/plugin-vue` are already installed in the workspace `devDependencies`, the existing UMD build at `src/umd/OdaComponents.js` already uses relative paths that are fully compatible with Vite's resolver, and Vite handles Vue SFC transformation natively without the Babel and `rollup-plugin-vue` chain currently used. The new config file `vite.config.lib.mjs` is added alongside the existing `rollup.config.mjs` so the original per-component build is not disturbed.

## 2. Architecture Overview

### 2.1 High-Level Flow
The new build pipeline adds a parallel build path alongside the existing Rollup pipeline. The entry point `src/umd/OdaComponents.js` re-exports every component, directive, service, and style object using relative paths (`../accordion/Accordion.vue`, etc.). Vite Library Mode reads this file, resolves all relative imports through `@vitejs/plugin-vue` for `.vue` files, and bundles the result into two output files: `dist/oda-components.mjs` (ES module) and `dist/oda-components.cjs` (CommonJS). `vite-plugin-dts` runs in parallel and rolls all TypeScript declarations into a single `dist/oda-components.d.ts` file. External dependencies (`vue`, `chart.js/auto`, `quill`) are not bundled. The internal packages `@oda-components/core`, `@oda-components/icons/baseicon`, and `@oda-components/icons/baseicon/style` are inlined into the bundle via Vite aliases that redirect their import specifiers to the actual source files on disk. The packages `@oda-components/styled` and `@oda-components/style-tokens` are resolved from their pre-built `dist/index.mjs` outputs via the normal node_modules resolution chain (they must be built first). After `vite build --config vite.config.lib.mjs` completes, consumers can write `import { OdaButton } from '@oda-components/components'` against the package root.

### 2.2 Component Diagram

```

        | import { OdaButton } from '@oda-components/components'

        ^

        |
        +-- vue          (external, peer dep, NOT bundled)

        +-- quill         (external, NOT bundled)
```

### TASK 2: Create vite.config.lib.mjs

**File**: `D:/VNPT_ODA/poc-folder/ODA_COMPONENT/oda-components/packages/components/vite.config.lib.mjs`
**Action**: CREATE
**Depends on**: TASK 1 must be completed first (vite-plugin-dts must be installed)

#### 2.1 Description
This is the core deliverable. It configures Vite in Library Mode targeting `src/umd/OdaComponents.js` as the single entry point. The `build.lib` options produce two output formats: `es` (`.mjs`) and `cjs` (`.cjs`). The `build.rollupOptions.external` array excludes `vue`, `chart.js/auto`, and `quill` from the bundle. `build.rollupOptions.output.inlineDynamicImports` is set to `true` to prevent Rollup from splitting the output into multiple chunks. `resolve.alias` mirrors the alias entries from `rollup.config.mjs` exactly so that `@oda-components/core/*` and `@oda-components/icons/baseicon*` are resolved to source files and inlined into the bundle. `build.cssCodeSplit` is set to `false` because the library uses a JS-based passthrough/token style system with no external CSS imports. The `dts` plugin is configured with `rollupTypes: true` to produce a single merged declaration file.

#### 2.2 Complete Implementation

Create the file at the exact path above with the following complete content. Copy every character exactly.

```javascript
// vite.config.lib.mjs
// Vite Library Mode — single-bundle build for @oda-components/components
// Entry: src/umd/OdaComponents.js
// Output: dist/oda-components.mjs, dist/oda-components.cjs, dist/oda-components.d.ts

import { dirname, resolve } from 'path';
import { fileURLToPath } from 'url';
import vue from '@vitejs/plugin-vue';
import dts from 'vite-plugin-dts';

const __dirname = dirname(fileURLToPath(import.meta.url));

/**
 * External packages: these are NOT inlined into the bundle.
 * Consumers must install these as peer dependencies.
 */
const EXTERNALS = [
    'vue',
    'chart.js/auto',
    'quill'
];

/**
 * Alias map: mirrors the alias entries in rollup.config.mjs exactly.
 * These redirect @oda-components/core/* and @oda-components/icons/baseicon*
 * imports to their source files so they are INLINED into the single bundle.
 *
 * @oda-components/styled and @oda-components/style-tokens are NOT aliased here;
 * they are resolved from their pre-built dist/index.mjs via node_modules.
 */
const ALIASES = [
    // core sub-paths (most specific first)
    {
        find: '@oda-components/core/api',
        replacement: resolve(__dirname, '../core/src/api/Api.js')
    },
    {
        find: '@oda-components/core/base/style',
        replacement: resolve(__dirname, '../core/src/base/style/BaseStyle.js')
    },
    {
        find: '@oda-components/core/base',
        replacement: resolve(__dirname, '../core/src/base/Base.js')
    },
    {
        find: '@oda-components/core/basecomponent/style',
        replacement: resolve(__dirname, '../core/src/basecomponent/style/BaseComponentStyle.js')
    },
    {
        find: '@oda-components/core/basecomponent',
        replacement: resolve(__dirname, '../core/src/basecomponent/BaseComponent.vue')
    },
    {
        find: '@oda-components/core/basedirective',
        replacement: resolve(__dirname, '../core/src/basedirective/BaseDirective.js')
    },
    {
        find: '@oda-components/core/baseeditableholder',
        replacement: resolve(__dirname, '../core/src/baseeditableholder/BaseEditableHolder.vue')
    },
    {
        find: '@oda-components/core/baseinput',
        replacement: resolve(__dirname, '../core/src/baseinput/BaseInput.vue')
    },
    {
        find: '@oda-components/core/config',
        replacement: resolve(__dirname, '../core/src/config/OdaComponents.js')
    },
    {
        find: '@oda-components/core/service',
        replacement: resolve(__dirname, '../core/src/service/OdaComponentsService.js')
    },
    {
        find: '@oda-components/core/useattrselector',
        replacement: resolve(__dirname, '../core/src/useattrselector/UseAttrSelector.js')
    },
    {
        find: '@oda-components/core/useid',
        replacement: resolve(__dirname, '../core/src/useid/UseId.js')
    },
    {
        find: '@oda-components/core/usestyle',
        replacement: resolve(__dirname, '../core/src/usestyle/UseStyle.js')
    },
    {
        find: '@oda-components/core/utils',
        replacement: resolve(__dirname, '../core/src/utils/Utils.js')
    },
    // core root (must come AFTER all sub-path aliases)
    {
        find: '@oda-components/core',
        replacement: resolve(__dirname, '../core/src/index.js')
    },
    // icons (only baseicon sub-paths are used by components)
    {
        find: '@oda-components/icons/baseicon/style',
        replacement: resolve(__dirname, '../icons/src/baseicon/style/BaseIconStyle.js')
    },
    {
        find: '@oda-components/icons/baseicon',
        replacement: resolve(__dirname, '../icons/src/baseicon/BaseIcon.vue')
    }
];

export default {
    plugins: [
        vue(),
        dts({
            // Path to the tsconfig used for type resolution
            tsconfigPath: resolve(__dirname, './tsconfig.json'),
            // Merge all .d.ts files into a single output file
            rollupTypes: true,
            // Output the merged declarations into the dist directory
            outDir: resolve(__dirname, './dist'),
            // Name the merged declarations file to match the bundle name
            insertTypesEntry: true,
            // Suppress verbose dts output during build
            logLevel: 'warn'
        })
    ],
    resolve: {
        alias: ALIASES
    },
    build: {
        // Output directory relative to this config file's location
        outDir: resolve(__dirname, './dist'),
        // Do not empty the dist directory so the existing UMD output is preserved
        emptyOutDir: false,
        // Disable CSS code splitting — the library uses a JS-based PT style system
        cssCodeSplit: false,
        // Generate source maps for the bundle
        sourcemap: true,
        lib: {
            // Single entry point — the UMD barrel file that uses relative imports
            entry: resolve(__dirname, './src/umd/OdaComponents.js'),
            // The name exposed on the global window object for UMD (unused here but required by Vite)
            name: 'OdaComponents',
            // Output file base name (without extension)
            // Vite will append .mjs and .cjs automatically
            fileName: 'oda-components',
            // Build both ES module and CommonJS formats
            formats: ['es', 'cjs']
        },
        rollupOptions: {
            external: EXTERNALS,
            output: {
                // Prevent code-splitting; everything goes into one file per format
                inlineDynamicImports: true,
                // Map external package names to their global variable names
                // (only used for IIFE/UMD formats, harmless for es/cjs)
                globals: {
                    vue: 'Vue',
                    'chart.js/auto': 'Chart',
                    quill: 'Quill'
                }
            },
            onwarn(warning, warn) {
                // Suppress circular dependency warnings that are known and harmless
                // in this component library (same suppression as rollup.config.mjs)
                if (warning.code === 'CIRCULAR_DEPENDENCY') {
                    return;
                }
                // Suppress "use client" directive warnings from Vue internals
                if (warning.code === 'MODULE_LEVEL_DIRECTIVE') {
                    return;
                }
                warn(warning);
            }
        }
    }
};
```

## 7. Risk and Rollback

### 7.1 Known Risks

- **Risk**: `@oda-components/styled` or `@oda-components/style-tokens` dist files do not exist. **Mitigation**: Run their build scripts (or verify them manually) before running `build:lib`. Both packages expose only a `dist/index.mjs` entry — if the file is missing, Vite will fail with `Error: Cannot find module '@oda-components/styled'`. See Section 5.2 for the check command.

- **Risk**: `@oda-components/utils` is a declared workspace dependency but has no `src/` or `dist/`. The `src/umd/OdaComponents.js` entry does NOT import `@oda-components/utils` directly (it uses the local `../utils/Utils.js` path), so this does not block the build. However, `@oda-components/core` may import `@oda-components/utils` internally. If the core package was built with a bundled copy of utils, this is not a problem. If it imports utils at runtime, the alias chain in `vite.config.lib.mjs` must be extended. **Mitigation**: Run `pnpm --filter @oda-components/core build` first (Step 1 of Section 5.1). If that succeeds, the utils dependency is already resolved inside the core build.

- **Risk**: Circular imports between components cause Rollup warnings even with `inlineDynamicImports: true`. **Mitigation**: The `onwarn` handler in `vite.config.lib.mjs` suppresses `CIRCULAR_DEPENDENCY` warnings. The bundle will still be produced correctly — circular deps in a component library that share a common base class are expected and harmless at runtime.

- **Risk**: `vite-plugin-dts` version 4.x may fail if `skipLibCheck: false` in `tsconfig.json` causes TypeScript errors in `.vue` source files. **Mitigation**: If the `dts` plugin throws, change `"skipLibCheck"` from `false` to `true` in `packages/components/tsconfig.json` and re-run. This suppresses type errors in third-party `.d.ts` files and is the standard setting for library builds.

- **Risk**: The `target: "es5"` in `tsconfig.json` may conflict with Vite's default `es2015` target. **Mitigation**: Vite does not read `tsconfig.json`'s `target` for transpilation — it uses `build.target` from `vite.config`. The default `build.target` in Vite 8 is `'modules'` (equivalent to `es2015`), which is appropriate for a library. No change to `vite.config.lib.mjs` is needed for this.

- **Risk**: `emptyOutDir: false` means stale files from previous builds accumulate in `dist/`. **Mitigation**: This is intentional to preserve `dist/umd/`. If a clean rebuild is needed, manually delete `dist/oda-components.*` files before running `build:lib`.

### 7.2 Rollback Procedure

If something breaks and needs to be reverted:

**Revert vite.config.lib.mjs** (delete the new file):
```bash
rm D:/VNPT_ODA/poc-folder/ODA_COMPONENT/oda-components/packages/components/vite.config.lib.mjs
```

**Revert packages/components/tsconfig.json** (change `"moduleResolution"` back to `"node"`):
Change the single line `"moduleResolution": "bundler"` back to `"moduleResolution": "node"` in `D:/VNPT_ODA/poc-folder/ODA_COMPONENT/oda-components/packages/components/tsconfig.json`.

**Revert packages/components/package.json** (three reverts):
1. Remove the `"build:lib": "vite build --config vite.config.lib.mjs"` line from `scripts`
2. Change `"main"` back to `"./src/index.js"`
3. Change `"module"` back to `"./src/index.js"`
4. Change `"types"` back to `"./src/index.d.ts"`
5. Remove the `"."` entry from the `exports` object

**Revert root package.json**:
Remove the `"build:lib:single": "pnpm --filter ./packages/components build:lib"` line from `scripts`.

**Verify original build still works** after rollback:
```bash
cd D:/VNPT_ODA/poc-folder/ODA_COMPONENT/oda-components/packages/components
pnpm run build
```
Expected: Rollup build completes with 300+ output files in `dist/` as before.
